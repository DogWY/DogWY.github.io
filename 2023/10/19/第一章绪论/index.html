第一章 绪论

1.1 什么是算法

算法: 解决问题的方法或过程, 严格地讲是满足下述性质的指令序列:

  • 输入: 有零个或多个外部量作为算法输入
  • 输出: 算法产生至少一个量作为输出
  • 确定性: 组成算法的每条指令清晰、无歧义
  • 有限性: 算法中每条指令的执行次数有限, 执行每条执行的时间也有限

程序: 算法用某种程序设计语言的具体实现.

程序可以不满足有限性

1.2 计算机问题求解

问题求解周期:

  1. 抽象: 对问题形式化描述
  2. 建模: 对问题和数据分析和建模
  3. 算法: 设计有效的计算机算法
  4. 实现: 用特定程序设计语言实现算法
  5. 验证: 测试、验证与推广

1.3 为什么学习算法

软件 = 算法 + 数据结构

1.4 渐进复杂性

算法复杂度是算法运行所需要的计算机资源的量，需要时间资源的量称为时间复杂性，需要的空间资源的量称为空间复杂性.

而在实际情况下, 影响算法执行时间的因素有很多, 比如:

  • 计算机配置
  • 问题规模
  • 输入实例

我们认为时间复杂度不应该是在特定计算机上求解某一个输入实例所需要的运行时间；而应该是一个不依赖于计算机配置、问题规模和输入实例的抽象表示。

此外, 我们在考虑算法的复杂度时, 也要全面考虑实际情况. 具体来说就是要综合考虑最好、最坏和平均的情况.

1.5 大O表示

O的定义

设\(f(N)\)和\(g(N)\)是定义在正数集上的正函数，如果存在正的常数\(C\)和自然数\(N_0\)，使得当\(N≥N_0\)时有\(f(N) ≤Cg(N)\)，则称函数\(f(N)\)当\(N\)充分大时上有界，\(g(N)\)是\(f(N)\)的一个上界，记为\(f(N)=O(g(N))\)，即\(f(N)\)的阶不高于\(g(N)\)的阶。

大O表示法的运算符规则

\[ \begin{align} &O(f) + O(g) = O(max(f,g))\\ &O(f) + O(g) = O(f+g)\\ &O(f) O(g) = O(fg)\\ &如果g(N) = O(f(N)),则O(f)+O(g)=O(f)\\ &O(Cf(N))=O(f(N)),其中C是一个正的常数\\ &f=O(f) \end{align} \]

大O比率定理

对于函数\(f(N)\)和\(g(N)\)，如果极限\(\lim\limits_{n\rightarrow \infty}f(N)/g(N)\)存在，则\(f(N) = O(g(N))\)当且仅当存在正的常数\(C\)，使得\(\lim\limits_{n\rightarrow \infty}f(N)/g(N) \le C\)

时间复杂度类别

\[ \begin{align*} O(1)<O(\log_n)<O(n)<O(n\log_n)<O(n^2)<\cdots<O(2^n)<O(n!)<O(n^n) \end{align*} \]
